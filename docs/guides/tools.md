# Shared tools

The Tools area is a curated directory of systems engineering teams use every day. Each entry has a name, description, optional icon, and one or more destinations.

Destinations can represent environments or distinct uses, for example:

- Development, staging, and production consoles;
- dashboards and operational views;
- repository, issue tracker, or documentation entry points.

## Open a destination

Open **Tools**, find the shared system, and choose the destination label that matches your task. Destinations open outside the portal.

## Add or change a tool

Administrators manage tools under **Settings → Tools**. Every tool requires at least one destination with a valid URL. Leaving the icon URL empty makes Perongen use a generated monogram consistent with the portal identity.

Tool configuration can also be provided in `perongen.yaml`; a saved control-plane override takes precedence until the section is reset.
